How it works

Three layers of automation

No servers, no databases, no manual work. ProjAI chains together browser automation, AI inference, and cloud infrastructure into a single autonomous pipeline.

01 Browse

Headless browser navigates booru platforms, handles auth flows and dynamic content

Puppeteer launches in stealth mode, accepts consent cookies, disables SFW toggles, and scrolls through paginated results — extracting image URLs from both Gelbooru and Rule34 simultaneously.

02 Think

Local LLM parses, validates and optimizes search tags in real-time

Qwen2.5-7B runs entirely on the Actions runner. It interprets natural language queries, cross-references against autocomplete APIs, and constructs optimal tag combinations for each source.

03 Deliver

Ngrok tunnels a live web interface accessible from anywhere for 6 hours

The Express server serves a Material You dark UI with search, visit buff, and AI chat tabs. Results stream in as Puppeteer discovers them — no waiting for the full crawl to finish.

Setup guide

Fork, configure, run.

Five minutes from fork to a running agent. No dependencies to install locally.

1

Fork the repository

Go to github.com/minhtristudio/nsfw-art-agent and click Fork. This creates your own copy to customize freely.

git clone https://github.com/YOUR_USER/nsfw-art-agent.git
2

Add GitHub Secrets

In your fork, go to Settings → Secrets → Actions and add:

NGROK_AUTH_TOKENYour auth token from ngrok.com
NGROK_DOMAINOptional — reserved static domain
3

Enable Actions & run

Open the Actions tab, enable workflows, then trigger deploy.yml. The workflow will set up Node.js, compile the LLM, download Qwen2.5-7B, launch Puppeteer, and start ngrok — all automatically.

Access the agent

Check the workflow logs for the ngrok output URL. It looks like https://xxxx.ngrok-free.app. Open it in your browser — the agent UI loads with search, visit buff, and AI chat ready to use.

5

Customize (optional)

Swap the LLM model in deploy.yml, change worker counts, adjust search sources — everything is configurable through environment variables and the web UI.

FAQ

Common questions

Is this free to run?

Yes. Everything runs on GitHub Actions free tier (2,000 min/month). Each session lasts up to 6 hours. You also need a free ngrok account for the tunnel — no paid plans required.

What are the requirements?

A GitHub account and a browser. The agent runs entirely on GitHub's infrastructure (2-core CPU, 7 GB RAM, 14 GB disk). No local hardware, no installations, no cloud accounts.

How long does a session last?

Up to 6 hours per workflow run. First run takes 3–5 extra minutes for the LLM model download. After that, subsequent runs start in under a minute if the cache is warm.

Can I use a different model?

Any GGUF model compatible with llama-cpp-python. Llama 3, Mistral, Yi — just change the download URL in deploy.yml. Keep in mind the runner has 7 GB RAM.

When will the repo go public?

We're cleaning up the codebase and writing documentation. Follow this page for updates — the repo will be open-sourced once everything is production-ready.
