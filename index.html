Setup Guide

How to Fork & Deploy

Follow these steps to set up your own instance of ProjAI once the repository goes open source.

01

Fork the Repository

Navigate to the nsfw-art-agent repository on GitHub and click the Fork button in the top-right corner. This creates a personal copy of the repository under your own GitHub account, giving you full control over the codebase while maintaining a link to the original project for upstream updates.

# After forking, clone your fork locally git clone https://github.com/YOUR_USERNAME/nsfw-art-agent.git cd nsfw-art-agent
02

Set Up GitHub Secrets

Go to your forked repository's Settings → Secrets and variables → Actions. Add the following required secrets that the GitHub Actions workflow needs to function properly. These secrets are used for ngrok authentication and any API keys needed for external services.

NGROK_AUTH_TOKEN Your ngrok authentication token from ngrok.com
NGROK_DOMAIN (Optional) Your reserved static ngrok domain for consistent URLs

To get a free ngrok auth token, sign up at ngrok.com/signup and navigate to Your Authtoken section.

03

Enable GitHub Actions

In your forked repository, navigate to the Actions tab. GitHub will prompt you to enable workflows — click "I understand my workflows, go ahead and enable them". The deploy workflow (deploy.yml) will automatically build the environment, download the LLM model, and start the server with ngrok tunneling.

04

Access Your Agent

Once the GitHub Actions workflow completes (usually 3-5 minutes on first run due to LLM download), it will output your ngrok URL in the workflow logs. Look for the "ngrok URL" step output — this is your public access URL. Click it to open the ProjAI web interface and start searching!

# Your agent URL will look like: https://xxxx-xxxx-xxxx.ngrok-free.app # The workflow runs for 6 hours before auto-stopping. # Manually re-trigger to restart.

GitHub Actions free tier allows up to 2,000 minutes/month. Each run uses ~360 minutes (6 hours). You get approximately 5 runs per month on the free plan.

05

Configure & Customize (Optional)

After your agent is running, you can customize its behavior. The web interface provides controls for search sources, AI chat settings, and visit buff configuration. For deeper customization, edit the server.js and workflow files in your fork. You can also modify the LLM model by changing the download URL in deploy.yml.

Change AI model (Qwen2.5-7B, Llama, Mistral, etc.)

FAQ

Frequently Asked

Is this project free to use?

Yes! ProjAI runs entirely on GitHub Actions free tier. You only need a free ngrok account for the tunnel. No paid services, no VPS, no cloud costs. Everything is serverless and automated through GitHub's infrastructure.

What are the system requirements?

None! That's the beauty of it. The agent runs on GitHub Actions runners (2-core CPU, 7GB RAM, 14GB disk). You don't need any local hardware. Just a GitHub account and a web browser to access the agent's interface.

How long does each session last?

Each GitHub Actions run lasts up to 6 hours (the maximum for the free tier). The workflow includes a keep-alive mechanism and will auto-stop when time expires. Simply re-trigger the workflow to start a new session. First-time setup takes 3-5 minutes extra for LLM model download.

Can I use a different LLM model?

Absolutely. The default is Qwen2.5-7B-Instruct (Q4_K_M quantization) for a balance of quality and resource usage. You can swap it for any GGUF model supported by llama-cpp-python — Llama 3, Mistral, Yi, etc. Just update the model URL in the deploy workflow.

When will the repository go public?

We're actively preparing the codebase for open source release — cleaning up code, adding documentation, and ensuring everything works out of the box for new users. Follow this page or star the repository to get notified when it goes public.
